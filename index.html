Lesson 2: Webpages, Documents, and Structures

Dev Tools and starting with the big picture

In this lesson I learned a new approach to building a webpage by starting with the big picture and breaking it down to smaller elements. This approach is useful for beginners, practical, and often how front end developers approach their work. Tips for using this approach include breaking down a current webpage and using it for your own purposes. Exploring HTML elements and CSS using Chromes Dev tools and Inspect element.

Structure

Browsers need several ingredients to serve up your webpage: HTML to provide the structure, CSS to style the page, and javascript to define the interactions on the page. DOM is the tree like structure of the webpage. The browser uses the rules of the DOM to render the html elements of the page.

Boxes

Boxes are essential to front end development and HTML. Every visual element on a web page is a box. This is pretty amazing. And implementing a design includes the first step of "Boxifying" the design.

CSS Reference Properties

Be prepared to get used to looking up various properties in the CSS Reference - its an important developer skill to use documentation. Reference here

CSS controls HTML but how?

Helpful explanation of how CSS controls HTML here.Specify a selector such as h1, body, or div class = "description". Next add curly braces and within these curly braces define a "rule" to be applied to that selector. This rule can also be called a declaration which is a property followed by a colon and then value.

CSS helps us avoid mistakes

Defining the style within the html document is error prone because you're writing a lot of the same code in multiple places. Defining style in CSS and applying it to a well structured HTML doc provides consistency, less bugs, and an easier way to experiment with format and style.

The Box Model = margin, border, padding, content

Use dev tools to explore the box model. Specifically the "Computed" tab is helpful. Content is the nucleas, Padding is impacted by the background color. The space around the content. Border is the properties of the box. Margin clears an area around the border. Space between boxes.

Element size calculation

Can set box size by percentage or pixels. Percentage resizes but pixels don't based on the screensize.

Positioning boxes

Use Flex box layout. Divs are block elements and to get them to appear next to eachother rather than their default full width of a page you can the rule diplay: flex; box
